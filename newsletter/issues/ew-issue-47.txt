** Ember and the future of the web
------------------------------------------------------------

An excellent write-up by Brandon Hays on how web development is changing and frameworks like Ember are positioned to turn the browser in to a real application runtime. Well worth a read. (http://frontside.io/blog/2014/02/24/ember-and-the-future-of-the-web.html)


** Ghost: &quot;Hello Ember&quot;
------------------------------------------------------------

The popular Open Source Blogging Platform, Ghost, will re-write their Backbone Admin UI using Ember. After [much discussion in the OS community](https://github.com/TryGhost/Ghost/issues/2144), the Ghost team explain why they chose Ember over Angular, React and Backbone + Marionette. (http://dev.ghost.org/hello-ember/)

** Getting started with Broccoli and Ember.js
------------------------------------------------------------

A walkthrough getting an Ember project up and building with the new Broccoli build tool. (http://ampersate.com/getting-started-with-broccoli-and-emberjs)


** A Common Resource Route Pattern in Ember.js
------------------------------------------------------------

A look at how to DRY up your routes setup in certain use cases. (http://balinterdi.com/2014/02/26/a-common-resource-route-pattern-in-ember-dot-js.html)


** Know your routes in Ember.js
------------------------------------------------------------

A post which illustrates the difference between the &#x60;&#x60;Application&#x60;&#x60; route and the &#x60;&#x60;ApplicationIndex&#x60;&#x60; route through a redirect example. (http://ugisozols.com/blog/2014/02/25/know-your-routes-in-emberjs/)


** Deploy an Ember.js App on Heroku
------------------------------------------------------------

An example of how to deploy an Ember application to Heroku using Node.js &amp; Express to server the app. (http://nepalonrails.com/blog/2014/03/deploy-emberjs-app-on-heroku/)


** Ember.js with a Separate Rails API
------------------------------------------------------------

This post shares thoughtbot&#x27;s experiences building an Ember application using Rails as the server-side API and a Rails project to manage the Ember app. (http://robots.thoughtbot.com/emberjs-with-a-separate-rails-api)


** Cross-browser AJAX uploads with Ember.js and mOxie
------------------------------------------------------------

A guide to using the mOxie library to support cross-browser uploads. ([demo](http://scribu.github.io/ember-moxie-demo/)) (http://scribu.net/blog/cross-browser-uploads-ember-moxie.html)
